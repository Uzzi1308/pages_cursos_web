Estructura básica de una página HTML

Objetivos de aprendizaje

  • Comprender cada sección clave del esqueleto HTML
  • Identificar qué hace cada etiqueta principal
  • Crear un documento HTML estructurado correctamente
  • Usar comentarios para identificar partes del HTML
  • Preparar archivos para agregar contenido real

¿Qué es la estructura básica de HTML?

Todo archivo HTML sigue una estructura mínima para que el navegador entienda cómo debe interpretar el documento. Esta estructura funciona como el marco donde luego se agregan textos, imágenes, botones, estilos y scripts.

Sin importar qué tan complejo sea un sitio web, todos comienzan con esta misma base. Desde la página de inicio de Google hasta una aplicación web sofisticada, todos respetan esta estructura fundamental.

Como los cimientos de una casa

Cimientos
Sin cimientos sólidos, no puedes construir nada encima. Lo mismo pasa con HTML.
Estructura
Las paredes y el techo definen los espacios. HTML define las secciones de tu página.
Contenido
Una vez que tienes la estructura, puedes llenarla con muebles y decoración.

Importante

No puedes empezar un proyecto profesional sin dominar esta base. Es como intentar construir un edificio sin planos: simplemente no funciona.

Partes del documento HTML

Cada documento HTML se compone de elementos específicos que cumplen funciones diferentes. Vamos a analizar cada uno en detalle:

<!DOCTYPE html>

¿Qué es? Una declaración que indica al navegador que el documento usa HTML5.

¿Para qué sirve? Le dice al navegador cómo interpretar el código. Sin esta línea, el navegador puede entrar en "modo quirks" (modo de compatibilidad antiguo) y renderizar la página de forma incorrecta.

Importante: No es una etiqueta HTML, es solo una declaración. Siempre debe ir en la primera línea del documento.

<html lang="es">

¿Qué es? La etiqueta raíz que contiene todo el documento.

El atributo lang: Indica el idioma principal del contenido. Esto ayuda a los lectores de pantalla (accesibilidad) y mejora el SEO.

Valores comunes: "es" para español, "en" para inglés, "fr" para francés, etc.

Regla: Todo el contenido del documento debe ir entre <html> y </html>.

<head>

¿Qué es? La sección de configuración del documento.

Contenido: Aquí van los metadatos, el título de la pestaña, enlaces a archivos CSS, scripts, favicons y más.

Importante: El contenido del <head> NO es visible para el usuario en la página. Solo configura cómo debe comportarse el documento.

Analogía: Es como el panel de control de un avión. El piloto lo necesita, pero los pasajeros no lo ven.

<body>

¿Qué es? La parte visible del documento.

Contenido: Todo lo que el usuario ve en la pantalla va aquí: textos, imágenes, menús, párrafos, botones, formularios, videos, etc.

Regla fundamental: Si quieres que algo aparezca en la página, debe estar dentro del <body>.

Metadatos esenciales del <head>

El <head> contiene información vital que prepara tu página para funcionar correctamente. Estas son las etiquetas más importantes:

<meta charset="UTF-8">

Codificación de caracteres

UTF-8 es el estándar universal de codificación que permite mostrar correctamente letras con acentos, la ñ, símbolos especiales y caracteres de múltiples idiomas.

Sin esta línea, los acentos y caracteres especiales podrían verse como símbolos raros (como � o ÃÃ±).

<meta name="viewport">

Meta viewport HTML 
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Responsive design

Esta etiqueta es fundamental para dispositivos móviles. Le dice al navegador que ajuste el ancho de la página al ancho del dispositivo.

width=device-width: El ancho de la página se adapta al ancho de la pantalla.
initial-scale=1.0: El zoom inicial es del 100%.

Sin esta etiqueta, tu página se vería diminuta en celulares.

<title>

Esta etiqueta controla el texto que aparece en la pestaña del navegador. Es lo primero que ven los usuarios cuando tienen múltiples pestañas abiertas, y también es importante para el SEO.

Ejemplo de título HTML 
<title>Mi primera página web</title>

Nota para después

En lecciones posteriores aprenderás a agregar hojas de estilo CSS en el <head> usando la etiqueta <link>.

Demostración de la estructura completa

Ahora que comprendes cada parte, veamos cómo se ve un documento HTML completo con todos sus elementos básicos:

Plantilla HTML5 completa HTML 
<!DOCTYPE html>
<html lang="es">

<head>
    <!-- Metadatos y configuración del documento -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- Título que aparece en la pestaña del navegador -->
    <title>Mi primera página</title>
</head>

<body>
    <!-- Contenido visible para el usuario -->
    <h1>Hola, esta es mi página</h1>
    <p>Este es mi primer párrafo en HTML.</p>
</body>

</html>

Estructura lista para usar

Esta es la plantilla que usarás en todos tus proyectos. Más adelante agregarás más contenido en el <body> y configuraciones en el <head>, pero la estructura base siempre será esta.

Errores comunes al iniciar

Estos son los errores más frecuentes que cometen los principiantes. Conocerlos te ayudará a evitar frustraciones:

Cerrar mal las etiquetas

Todas las etiquetas que se abren deben cerrarse. Por cada <div> debe haber un </div>. El orden también importa.

Contenido en el lugar equivocado

Error común: poner textos o imágenes dentro del <head>. Todo contenido visible debe ir en el <body>.

Olvidar el DOCTYPE

Sin el <!DOCTYPE html> al inicio, el navegador puede interpretar tu código de forma incorrecta o entrar en modo quirks.

No especificar el idioma

Olvidar el atributo lang="es" en la etiqueta <html> afecta la accesibilidad y el SEO de tu sitio.

No indentar el código

Un código sin indentación es difícil de leer y mantener. Usa tabulaciones o espacios para organizar visualmente tu HTML.

Confiar en que el navegador arregle errores

Los navegadores modernos intentan corregir HTML mal escrito, pero eso NO es buena práctica. Escribe HTML correcto desde el inicio.

Conceptos complementarios

¿Qué es la indentación y por qué es importante?

La indentación (también llamada sangría) es el espacio que se agrega al inicio de una línea de código para mostrar la jerarquía y estructura del documento.

Código sin indentar (difícil de leer) HTML 
<html>
<head>
<title>Mi página</title>
</head>
<body>
<h1>Título</h1>
<p>Párrafo</p>
</body>
</html>
Código indentado correctamente (fácil de leer) HTML 
<html>
  <head>
    <title>Mi página</title>
  </head>
  <body>
    <h1>Título</h1>
    <p>Párrafo</p>
  </body>
</html>

Beneficios de la indentación

  • Facilita la lectura y comprensión del código
  • Permite detectar errores más rápidamente
  • Es un estándar profesional en desarrollo web
  • Hace el trabajo en equipo más eficiente

Los navegadores son tolerantes (pero tú no deberías serlo)

Los navegadores modernos intentan interpretar HTML incluso si tiene errores. Pueden cerrar etiquetas automáticamente o corregir estructuras mal formadas.

Esto NO es una buena práctica

Aunque el navegador pueda mostrar tu página con errores, esto puede causar:

  • Comportamiento inconsistente entre diferentes navegadores
  • Problemas de rendimiento
  • Dificultad para mantener y escalar el proyecto
  • Problemas con herramientas de desarrollo y frameworks

Siempre escribe HTML válido y bien estructurado desde el principio.

Elementos de bloque vs elementos en línea

Este es un concepto que veremos en profundidad en lecciones posteriores, pero es bueno que lo conozcas desde ahora:

Diferencias básicas

  • Elementos de bloque: Ocupan todo el ancho disponible y comienzan en una nueva línea. Ejemplos: <div>, <p>, <h1>
  • Elementos en línea: Solo ocupan el espacio necesario y permiten que otros elementos estén en la misma línea. Ejemplos: <span>, <a>, <strong>

No te preocupes por memorizar esto ahora

En las próximas lecciones profundizaremos en este tema con ejemplos prácticos. Por ahora, solo necesitas saber que existen dos tipos principales de elementos.

Actividad práctica: Reconstruye una página HTML desde cero

Es momento de reforzar lo aprendido creando tu propio documento HTML sin copiar y pegar. Esto te ayudará a desarrollar memoria muscular y confianza:

  1. Crea un nuevo archivo llamado pagina2.html en tu carpeta de trabajo
  2. Escribe la estructura completa de HTML sin copiar y pegar. Escríbela tú mismo de memoria
  3. Cambia el título de la página por algo personalizado, por ejemplo: "Mi Segunda Página - [Tu Nombre]"
  4. Dentro del <body>, añade:
    • Un <h1> con un título principal
    • Un <p> con un párrafo descriptivo
  5. Comenta cada sección principal usando comentarios HTML:
    <!-- Este es el cuerpo de mi documento -->
  6. Guarda el archivo y ábrelo en tu navegador para verificar que funciona
  7. Abre las DevTools de Chrome (F12) e inspecciona la estructura de tu página

Objetivo de esta actividad

El objetivo no es solo crear la página, sino que puedas escribir la estructura HTML de memoria. Esto te dará confianza para futuros proyectos.

Consejo profesional

Muchos editores como VS Code tienen atajos para generar la estructura básica. Por ejemplo, escribe ! y presiona TAB. Pero es importante que primero aprendas a hacerlo manualmente para entender cada parte.

Resumen de la lección

Un documento HTML se compone de cuatro elementos fundamentales: la declaración <!DOCTYPE html>, la etiqueta raíz <html>, la sección de configuración <head> y el contenido visible <body>.

Cada sección cumple una función específica que el navegador respeta. El <head> contiene metadatos como la codificación UTF-8, la configuración viewport para responsividad y el título de la página.

Todo el contenido visible para el usuario debe ir dentro del <body>. Es fundamental mantener un HTML bien indentado y evitar errores comunes como cerrar mal las etiquetas o colocar contenido en el lugar equivocado.

Esta estructura es la base de cualquier sitio web profesional. Dominarla te permitirá construir páginas más complejas con confianza, sabiendo que tu código está bien fundamentado desde el inicio.
