Formularios Básicos

Objetivos de aprendizaje

  • Comprender la estructura básica de un formulario HTML
  • Crear formularios con diferentes tipos de inputs
  • Implementar validación básica con atributos HTML
  • Usar etiquetas y agrupaciones para formularios accesibles
  • Crear formularios de contacto y registro funcionales
  • Entender la diferencia entre GET y POST

Comunicación con formularios

Los formularios son el puente entre tus usuarios y tu aplicación web. Permiten recopilar información, autenticar usuarios, realizar búsquedas y mucho más. Sin formularios, la web sería solo de lectura.

Estructura básica: <form>

Todo formulario comienza con la etiqueta <form> que define el área interactiva y cómo se enviarán los datos.

<form>
  • Contenedor principal
  • Define el formulario
  • Atributos: action, method
action
  • URL que procesa los datos
  • Puede ser absoluta o relativa
  • Ej: "procesar.php"
method
  • GET (datos visibles)
  • POST (datos ocultos)
  • Por defecto: GET
Sintaxis básica de formulario HTML 
<form action="procesar.php" method="POST">
  <!-- Elementos del formulario -->
</form>

GET vs POST: Usa GET para búsquedas (datos visibles en URL). Usa POST para envío de datos sensibles (contraseñas, información personal).

El elemento <input>

<input> es el elemento más versátil de los formularios. Su apariencia y comportamiento cambian según el atributo type.

<input type="text">
<input type="email">
<input type="password">
<input type="number">
Tipos comunes de input HTML 
<!-- Texto simple -->
<input type="text" name="nombre" placeholder="Tu nombre">

<!-- Email con validación -->
<input type="email" name="email" required>

<!-- Contraseña oculta -->
<input type="password" name="password">

<!-- Número con rango -->
<input type="number" name="edad" min="0" max="120">

<!-- Fecha -->
<input type="date" name="fecha_nacimiento">

<!-- Botón de envío -->
<input type="submit" value="Enviar">

Selecciones: Checkbox y Radio

¿Cuándo usar cada uno?

Checkbox
  • Selección múltiple
  • Cada opción independiente
  • Ideal para preferencias
  • Ej: Intereses (Deportes, Música, Cine)
Radio Buttons
  • Selección única
  • Opciones mutuamente excluyentes
  • Mismo atributo name
  • Ej: Género (Masculino, Femenino, Otro)
Ejemplo: Checkbox vs Radio HTML 
<!-- CHECKBOX (múltiple selección) -->
<p>¿Qué deportes te gustan?</p>
<input type="checkbox" name="deportes" value="futbol" id="futbol">
<label for="futbol">Fútbol</label><br>

<input type="checkbox" name="deportes" value="baloncesto" id="baloncesto">
<label for="baloncesto">Baloncesto</label><br>

<!-- RADIO (selección única) -->
<p>¿Cuál es tu género?</p>
<input type="radio" name="genero" value="masculino" id="masculino">
<label for="masculino">Masculino</label><br>

<input type="radio" name="genero" value="femenino" id="femenino">
<label for="femenino">Femenino</label>

Regla de oro

Checkbox: Mismo name para agrupar conceptualmente, pero cada uno puede tener valor diferente.
Radio: Mismo name para hacerlos mutuamente excluyentes, diferente value para identificar la selección.

Más elementos: Textarea, Select, Label

<textarea>

Para texto multilínea (comentarios, mensajes).

<textarea name="mensaje" rows="4" cols="50">
Escribe tu mensaje aquí...
</textarea>

<select> y <option>

Listas desplegables para opciones predefinidas.

<select name="pais">
  <option value="mx">México</option>
  <option value="es">España</option>
  <option value="ar">Argentina</option>
</select>

<label>

Asocia texto descriptivo con elementos de formulario.

<!-- Método 1: Envolviendo -->
<label>
  <input type="checkbox" name="terminos">
  Acepto los términos
</label>

<!-- Método 2: Con atributo for -->
<input type="text" id="nombre">
<label for="nombre">Nombre completo:</label>

¡Etiquetas son esenciales!

Los elementos <label> no son solo decoración. Mejoran la accesibilidad (lectores de pantalla), permiten hacer clic en el texto para activar el input, y mejoran la usabilidad en dispositivos táctiles.

Validación básica con atributos

HTML5 introdujo validación nativa en el navegador, reduciendo la necesidad de JavaScript para validaciones simples.

required

Campo obligatorio

<input required>

minlength / maxlength

Longitud mínima/máxima

<input minlength="3" maxlength="50">

min / max

Valor mínimo/máximo

<input type="number" min="1" max="100">

type="email"

Validación de email

<input type="email">
Ejemplo de formulario validado HTML 
<form>
  <label for="usuario">Usuario:</label>
  <input type="text" id="usuario" name="usuario" 
         required minlength="3" maxlength="20">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="edad">Edad:</label>
  <input type="number" id="edad" name="edad" 
         min="18" max="99" required>

  <input type="submit" value="Registrarse">
</form>

Validación del lado del cliente ≠ Seguridad

La validación HTML5 es para mejorar la experiencia del usuario, no para seguridad. Los datos siempre deben validarse también en el servidor, ya que los usuarios pueden desactivar JavaScript o modificar el HTML.

Organizando formularios: <fieldset> y <legend>

Para formularios largos, es útil agrupar elementos relacionados. Esto mejora la accesibilidad y la usabilidad.

Información Personal
Preferencias
Código de agrupación HTML 
<form>
  <fieldset>
    <legend>Información Personal</legend>
    
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email">
  </fieldset>

  <fieldset>
    <legend>Preferencias</legend>
    
    <input type="checkbox" id="noticias" name="noticias">
    <label for="noticias">Recibir noticias</label>
    
    <input type="checkbox" id="ofertas" name="ofertas">
    <label for="ofertas">Recibir ofertas</label>
  </fieldset>
</form>

Beneficios de la agrupación

  • Accesibilidad: Los lectores de pantalla anuncian los grupos
  • Usabilidad: Los usuarios comprenden mejor la estructura
  • Estilo: Más fácil aplicar estilos CSS a grupos
  • Organización: Código más limpio y mantenible

Ejemplos prácticos de formularios

Contacto

Formulario de Contacto HTML 
<form action="contacto.php" method="POST">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="mensaje">Mensaje:</label>
  <textarea id="mensaje" name="mensaje" rows="4" required></textarea>

  <input type="submit" value="Enviar mensaje">
</form>

Registro

Formulario de Registro HTML 
<form action="registro.php" method="POST">
  <label for="usuario">Usuario:</label>
  <input type="text" id="usuario" name="usuario" 
         minlength="3" required>

  <label for="password">Contraseña:</label>
  <input type="password" id="password" name="password" 
         minlength="6" required>

  <label>Género:</label>
  <input type="radio" id="masculino" name="genero" value="m">
  <label for="masculino">Masculino</label>

  <input type="radio" id="femenino" name="genero" value="f">
  <label for="femenino">Femenino</label>

  <label for="pais">País:</label>
  <select id="pais" name="pais">
    <option value="mx">México</option>
    <option value="es">España</option>
    <option value="ar">Argentina</option>
  </select>

  <input type="submit" value="Crear cuenta">
</form>

Búsqueda

Formulario de Búsqueda HTML 
<form action="buscar.php" method="GET">
  <label for="busqueda">Buscar:</label>
  <input type="search" id="busqueda" name="q" 
         placeholder="¿Qué estás buscando?">

  <label for="categoria">Categoría:</label>
  <select id="categoria" name="categoria">
    <option value="todos">Todos</option>
    <option value="web">Web</option>
    <option value="imagenes">Imágenes</option>
  </select>

  <input type="submit" value="Buscar">
</form>

Nota sobre method

Observa que el formulario de búsqueda usa GET (los datos van en la URL), mientras que contacto y registro usan POST (datos ocultos). GET es ideal para búsquedas porque permite compartir la URL con los resultados.

Buenas prácticas para formularios

Accesibilidad

  • Siempre usa <label>
  • Orden lógico de tabulación
  • Agrupa con <fieldset>
  • Mensajes de error claros

Responsive

  • Inputs al 100% en móviles
  • Tamaños de toque adecuados
  • Usa tipos de input apropiados
  • Evita tablas en formularios

Usabilidad

  • Mínimo de campos necesarios
  • Placeholders como ejemplos
  • Validación en tiempo real
  • Botones con texto claro

Regla de oro: Menos es más

Cada campo adicional reduce la tasa de conversión. Pregúntate: ¿Realmente necesito este dato? Elimina campos innecesarios, agrupa información relacionada y usa tipos de input apropiados para cada dato.

Actividad práctica: Crea tus formularios básicos

Es hora de aplicar lo aprendido creando dos formularios funcionales usando solo HTML.

  1. Crea un archivo llamado contacto.html con este contenido base: 
    <!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Formulario de Contacto</title>
</head>
<body>
  <h1>Contáctanos</h1>
  <!-- Tu formulario irá aquí -->
</body>
</html>
  2. Dentro del body, crea un formulario de contacto con:
    • Campo "Nombre" (text, required)
    • Campo "Email" (email, required)
    • Campo "Asunto" (text)
    • Campo "Mensaje" (textarea, rows=5, required)
    • Checkbox: "Deseo recibir noticias"
    • Botón "Enviar mensaje" (type="submit")
    • Todos los campos deben tener <label> asociados
  3. Crea un segundo archivo registro.html con un formulario de registro:
    • Campo "Usuario" (text, minlength=3, required)
    • Campo "Email" (email, required)
    • Campo "Contraseña" (password, minlength=6, required)
    • Campo "Confirmar contraseña" (password, required)
    • Radio buttons para "Género" (Masculino, Femenino, Otro)
    • Select para "País" (3 opciones mínimas)
    • Checkbox: "Acepto términos y condiciones" (required)
    • Dos botones: "Registrarse" (submit) y "Limpiar" (reset)
  4. Revisa tus formularios:
    • ¿Todos los inputs tienen atributo name?
    • ¿Los labels usan atributo for correctamente?
    • ¿Los radio buttons del mismo grupo tienen mismo name?
    • ¿Los formularios se ven bien al abrirlos en el navegador?
    • ¿La validación funciona (intenta enviar campos vacíos)?

Consejo profesional

Usa el atributo autocomplete para ayudar a los navegadores: autocomplete="name", autocomplete="email", etc. Esto mejora la experiencia del usuario permitiendo autocompletar campos comunes.

Resumen de la lección

Los formularios HTML (<form>) permiten la interacción con los usuarios. Usan el atributo method (GET para datos visibles en URL, POST para datos ocultos) y action para especificar dónde enviar los datos.

El elemento <input> es el más versátil, cambiando su comportamiento según el type (text, email, password, number, checkbox, radio, submit, etc.). Otros elementos importantes son <textarea> (texto multilínea), <select> (listas desplegables) y <label> (etiquetas descriptivas).

HTML5 introdujo validación nativa con atributos como required, minlength, maxlength, min, max, y tipos especiales como email que validan automáticamente.

Para formularios complejos, usa <fieldset> y <legend> para agrupar elementos relacionados. Siempre prioriza la accesibilidad usando etiquetas correctamente asociadas y orden lógico de tabulación.
