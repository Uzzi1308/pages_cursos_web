Header, Nav, Main, Footer

Los elementos estructurales fundamentales para crear páginas web semánticas y accesibles.

Objetivos de aprendizaje

  • Comprender la función específica de cada elemento estructural
  • Aprender a usar correctamente <header> en diferentes contextos
  • Dominar el uso de <nav> para navegación principal y secundaria
  • Entender el papel único de <main> y las reglas para su uso
  • Implementar <footer> para pies de página y pies de sección
  • Crear una estructura completa y semánticamente correcta

Los pilares de la estructura web

En la lección anterior aprendiste sobre HTML semántico. Ahora profundizaremos en los cuatro elementos estructurales más importantes: <header>, <nav>, <main> y <footer>. Estos son los pilares sobre los que construyes cualquier página web moderna.

Estos elementos no solo mejoran la estructura, sino también la accesibilidad y el SEO.

El elemento <header>

¿Qué es un <header>?

Representa un grupo de contenido introductorio o de navegación. Puede contener elementos como logotipos, títulos, menús de navegación, formularios de búsqueda, etc.

Usos principales:

  • Header de página: La cabecera principal del sitio
  • Header de sección: Cabecera dentro de <article> o <section>
  • Header de artículo: Título, autor y fecha de publicación
Ejemplo de header HTML 
<header role="banner">
  <h1>Mi Sitio Web</h1>
  <p>El mejor lugar para aprender HTML</p>
  <nav aria-label="Navegación principal">
    <!-- Menú de navegación -->
  </nav>
</header>

¡Puedes tener múltiples headers!

A diferencia de <main>, puedes tener varios elementos <header> en una página: uno principal para toda la página, y otros dentro de artículos o secciones. Cada uno representa la cabecera de su sección contenedora.

El elemento <nav>

Tipos de navegación

Navegación Principal

Enlaces esenciales para navegar el sitio.

  • Normalmente en el <header>
  • Incluye: Inicio, Acerca, Servicios, Contacto
  • Esencial para la usabilidad
Navegación Secundaria

Enlaces complementarios o de pie de página.

  • Normalmente en el <footer>
  • Incluye: Términos, Privacidad, Mapa del sitio
  • No esencial pero útil
Breadcrumbs

Ruta de navegación jerárquica.

  • Muestra la ubicación actual
  • Formato: Inicio > Categoría > Subcategoría
  • Mejora la experiencia de usuario
Ejemplo completo de navegación HTML 
<!-- Navegación principal -->
<nav aria-label="Navegación principal" role="navigation">
  <ul>
    <li><a href="index.html" aria-current="page">Inicio</a></li>
    <li><a href="servicios.html">Servicios</a></li>
    <li><a href="contacto.html">Contacto</a></li>
  </ul>
</nav>

<!-- Breadcrumbs -->
<nav aria-label="Ruta de navegación">
  <ol>
    <li><a href="index.html">Inicio</a></li>
    <li><a href="blog.html">Blog</a></li>
    <li aria-current="page">Artículo actual</li>
  </ol>
</nav>

¿Cuándo NO usar <nav>?

No todos los grupos de enlaces merecen un <nav>. Por ejemplo:
NO usar nav: Enlaces sociales en el footer, enlaces patrocinados, enlaces dentro de un artículo.
SÍ usar nav: Menú principal, breadcrumbs, navegación entre páginas de un tutorial.

La regla más importante de HTML5

Solo debe haber UN elemento <main> por página.

¿Qué va dentro de <main>?

  • Contenido directamente relacionado con el tema central de la página
  • Contenido que es único en esa página (no se repite en otras)
  • Contenido que debería ser el foco para el usuario
  • Artículos, secciones, formularios principales

¿Qué NO va dentro de <main>?

  • Encabezados de página (<header>)
  • Navegación principal (<nav>)
  • Pies de página (<footer>)
  • Información complementaria (<aside>) que no es parte del flujo principal
  • Barras laterales de navegación
Uso correcto de main HTML 
<body>
  <header role="banner">...</header>
  
  <main role="main" aria-label="Contenido principal">
    <h1>Título principal de la página</h1>
    <p>Este es el contenido único y principal...</p>
    <!-- Más contenido principal -->
  </main>
  
  <footer role="contentinfo">...</footer>
</body>

¡Error común!

Muchos desarrolladores principiantes ponen TODO el contenido dentro de <main>. Esto es incorrecto. Solo el contenido principal y único debe ir dentro. Los headers, footers y navegación van fuera.

Política de privacidad

¿Qué es un <footer>?

Representa el pie de una sección o página. Normalmente contiene información sobre el autor, enlaces relacionados, información de copyright, etc.

Contenido típico:

  • Información de copyright (© 2024 Mi Empresa)
  • Información de contacto
  • Enlaces a redes sociales
  • Enlaces legales (Términos, Privacidad)
  • Mapa del sitio
  • Información del autor
  • Suscripción a newsletter
Ejemplo de footer HTML 
<footer role="contentinfo">
  <p>© 2024 Mi Sitio Web. Todos los derechos reservados.</p>
  <nav aria-label="Navegación del pie de página">
    <a href="privacidad.html">Política de Privacidad</a>
    <a href="terminos.html">Términos de Servicio</a>
  </nav>
  <address>
    Contacto: <a href="mailto:info@ejemplo.com">info@ejemplo.com</a>
  </address>
</footer>

Footers anidados

Al igual que <header>, puedes tener múltiples elementos <footer>. Puedes tener un footer principal para toda la página, y footers dentro de <article> o <section> para información específica de esa sección.

Jerarquía y anidamiento correcto

La relación entre estos elementos no es aleatoria. Siguen una jerarquía lógica que determina cómo deben anidarse para crear una estructura válida y semántica.

<body>
<header>
<nav>
<main>
<article>
<header>
<footer>

Reglas de anidamiento

  • <nav> puede ir dentro de <header> o <footer>
  • <header> y <footer> pueden ir dentro de <article> o <section>
  • <main> NO debe ir dentro de otros elementos de sección
  • Puede haber múltiples <header>, <nav> y <footer>
  • Solo puede haber un <main> por página
  • <main> debe ser hijo directo de <body>
Estructura completa válida HTML 
<body>
  <header role="banner">
    <h1>Mi Sitio</h1>
    <nav aria-label="Navegación principal">...</nav>
  </header>
  
  <main role="main">
    <article>
      <header>
        <h2>Título del artículo</h2>
      </header>
      <p>Contenido...</p>
    </article>
  </main>
  
  <footer role="contentinfo">
    <p>© 2024</p>
    <nav aria-label="Navegación secundaria">...</nav>
  </footer>
</body>

Código completo de la página HTML 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>MiBlog - HTML Semántico</title>
</head>
<body>
  <!-- HEADER PRINCIPAL -->
  <header role="banner">
    <h1>MiBlog</h1>
    <p>Aprendiendo desarrollo web</p>
    
    <!-- NAVEGACIÓN PRINCIPAL -->
    <nav aria-label="Navegación principal" role="navigation">
      <ul>
        <li><a href="index.html" aria-current="page">Inicio</a></li>
        <li><a href="articulos.html">Artículos</a></li>
        <li><a href="acerca.html">Acerca</a></li>
        <li><a href="contacto.html">Contacto</a></li>
      </ul>
    </nav>
  </header>

  <!-- CONTENIDO PRINCIPAL -->
  <main role="main" aria-label="Contenido principal">
    <article>
      <header>
        <h2>Introducción a HTML Semántico</h2>
        <p>Publicado el <time datetime="2024-01-15">15 de enero, 2024</time></p>
      </header>
      
      <p>El HTML semántico es fundamental para crear sitios web accesibles...</p>
      
      <section>
        <h3>Beneficios principales</h3>
        <p>Mejora la accesibilidad, el SEO y la mantenibilidad del código.</p>
      </section>
      
      <section>
        <h3>Elementos clave</h3>
        <p>Los elementos <code>&lt;header&gt;</code>, <code>&lt;nav&gt;</code>,
        <code>&lt;main&gt;</code> y <code>&lt;footer&gt;</code> son esenciales.</p>
      </section>
    </article>
    
    <aside aria-label="Categorías del blog">
      <h3>Categorías</h3>
      <ul>
        <li><a href="#">HTML</a></li>
        <li><a href="#">CSS</a></li>
        <li><a href="#">JavaScript</a></li>
      </ul>
    </aside>
  </main>

  <!-- FOOTER PRINCIPAL -->
  <footer role="contentinfo">
    <p>© 2024 MiBlog. Todos los derechos reservados.</p>
    
    <!-- NAVEGACIÓN SECUNDARIA -->
    <nav aria-label="Navegación del pie de página">
      <a href="privacidad.html">Política de Privacidad</a>
      <a href="terminos.html">Términos de Servicio</a>
      <a href="contacto.html">Contacto</a>
    </nav>
    
    <address>
      Escríbenos a: <a href="mailto:info@miblog.com">info@miblog.com</a>
    </address>
  </footer>
</body>
</html>

Estructura validada

Este código pasa todas las validaciones HTML5. Observa cómo:
1. Hay solo un <main>
2. Los <nav> están correctamente ubicados y etiquetados
3. Los <header> y <footer> están en sus contextos apropiados
4. La jerarquía de encabezados es correcta (<h1> → <h2> → <h3>)
5. Todos los elementos tienen roles ARIA apropiados

Accesibilidad y SEO

Accesibilidad Mejorada

  • Los lectores de pantalla pueden navegar por landmarks
  • Los usuarios pueden saltar al contenido principal fácilmente
  • Mejor experiencia para usuarios con discapacidades
  • Compatibilidad con tecnologías de asistencia

SEO Optimizado

  • Los motores de búsqueda entienden mejor la estructura
  • Mejor indexación del contenido principal
  • Mayor relevancia en resultados de búsqueda
  • Mejor experiencia de usuario (métrica SEO importante)

Mantenibilidad

  • Código más legible y organizado
  • Fácil de actualizar y modificar
  • Mejor colaboración entre desarrolladores
  • Compatibilidad futura garantizada

Atributos ARIA para mejorar accesibilidad

Atributos ARIA recomendados HTML 
<!-- Elementos semánticos con roles implícitos -->
<nav aria-label="Navegación principal">
  <!-- menú -->
</nav>

<main aria-label="Contenido principal">
  <!-- contenido -->
</main>

<header role="banner">
  <!-- cabecera -->
</header>

<footer role="contentinfo">
  <!-- pie -->
</footer>

<!-- Para elementos no semánticos que funcionan como landmarks -->
<div role="navigation" aria-label="Menú secundario">
  <!-- navegación secundaria -->
</div>

Nota: Los elementos semánticos como <nav>, <main>, <header> y <footer> ya tienen roles ARIA implícitos. Solo debes añadir roles explícitos si necesitas soporte para navegadores muy antiguos, o si usas elementos no semánticos (como <div>) para estas funciones.

Casos especiales y errores comunes

ERROR: Múltiples <main>

<body>
  <main>
    <h1>Sección 1</h1>
  </main>
  
  <main>  <!-- ¡ERROR! Solo un main permitido -->
    <h2>Sección 2</h2>
  </main>
</body>

Solución: Usa <section> o <article> para agrupar contenido dentro de un único <main>.

ERROR: <nav> para enlaces no esenciales

<article>
  <p>Lee más sobre <a href="#">HTML</a>.</p>
  
  <nav>  <!-- ¡ERROR! Enlaces no esenciales -->
    <a href="#">Enlace 1</a>
    <a href="#">Enlace 2</a>
  </nav>
</article>

Solución: Solo usa <nav> para bloques de navegación principales. Para enlaces dentro de contenido, usa un simple <div> o <p>.

ERROR: Header/footer en ubicación incorrecta

<body>
  <main>
    <header>  <!-- ¡ADVERTENCIA! Mejor fuera del main -->
      <h1>Título</h1>
    </header>
  </main>
</body>

Solución: El <header> principal debe ir antes de <main>. Los headers dentro de <main> deben estar dentro de <article> o <section>.

Casos especiales válidos

Múltiples <nav> en diferentes ubicaciones

Es perfectamente válido tener varios elementos <nav> en diferentes partes de la página:

  • Uno en el <header> para navegación principal
  • Uno en el <footer> para enlaces legales y secundarios
  • Uno en un <aside> para navegación dentro de una sección

<header> y <footer> dentro de <article>

Cada <article> puede tener su propio <header> (con título, autor, fecha) y <footer> (con información del artículo, enlaces relacionados).

Resumen de la lección

Los elementos <header>, <nav>, <main> y <footer> son los pilares fundamentales del HTML semántico moderno. Cada uno cumple una función específica y debe usarse de manera apropiada para crear páginas web bien estructuradas.

<header>

Contenido introductorio o de navegación. Puede aparecer múltiples veces.

<nav>

Bloques de navegación importantes. Usar solo para navegación esencial.

<main>

Contenido principal único. Solo uno por página.

<footer>

Pie de página o sección. Información de cierre y enlaces relacionados.

Beneficios clave
  • Accesibilidad: Mejora la navegación con lectores de pantalla
  • SEO: Los motores de búsqueda entienden mejor tu contenido
  • Mantenibilidad: Código más organizado y fácil de entender
  • Compatibilidad: Funciona en todos los navegadores modernos

Recuerda la regla más importante: solo un <main> por página, y úsalo solo para el contenido verdaderamente principal y único.
