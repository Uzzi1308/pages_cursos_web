Section, Article, Aside

Elementos de sección semántica en HTML5

Objetivos de aprendizaje

  • Diferenciar entre <section>, <article> y <aside>
  • Comprender el uso específico de cada elemento
  • Aplicar correctamente la jerarquía de encabezados
  • Crear esquemas de documento semánticamente correctos
  • Implementar estructuras anidadas apropiadas
  • Mejorar SEO y accesibilidad con secciones semánticas

Los Tres Pilares de la Sección Semántica

En HTML5, tenemos tres elementos fundamentales para estructurar el contenido de forma semántica: <section>, <article> y <aside>. Cada uno tiene un propósito específico que ayuda a navegadores, motores de búsqueda y tecnologías de asistencia a entender la organización y relación del contenido.

<section>

Agrupación Temática

Agrupa contenido relacionado temáticamente que forma parte de un todo mayor.

Ejemplo: Capítulos de un libro, pestañas de una aplicación

<article>

Contenido Independiente

Contenido autocontenido que podría distribuirse independientemente.

Ejemplo: Post de blog, artículo de noticias, comentario

<aside>

Contenido Relacionado

Contenido tangencialmente relacionado con el contenido principal.

Ejemplo: Sidebar, notas al margen, publicidad relacionada

¿Por qué usar estos elementos?

Cada elemento crea un esquema semántico que ayuda a los lectores de pantalla a navegar el contenido, mejora el SEO al indicar a los motores de búsqueda la estructura del contenido, y hace el código más mantenible y comprensible para otros desarrolladores.

<section> - Agrupación Temática

El elemento <section> representa una sección genérica de un documento. Agrupa contenido temáticamente relacionado. Cada <section> debe ser identificado, típicamente incluyendo un elemento de encabezado (<h1>-<h6>) como hijo del elemento <section>.

Uso correcto de <section> HTML 
<!-- Una página de servicios -->
<main>
  <h1>Nuestros Servicios</h1>
  
  <section>
    <h2>Diseño Web</h2>
    <p>Creación de sitios web modernos y responsivos...</p>
  </section>
  
  <section>
    <h2>Desarrollo de Apps</h2>
    <p>Desarrollo de aplicaciones móviles nativas...</p>
  </section>
  
  <section>
    <h2>SEO y Marketing</h2>
    <p>Optimización para motores de búsqueda...</p>
  </section>
</main>

No usar <section> como contenedor de estilo

Mal uso: <section> solo para aplicar CSS.
Buen uso: <section> para agrupar contenido temático.
Si necesitas un contenedor solo para estilos, usa <div>.

Casos de uso apropiados para <section>:

  • Capítulos de un libro o artículo largo
  • Pestañas en una interfaz de usuario
  • Secciones temáticas en una página (servicios, equipo, testimonios)
  • Introducción y conclusión de un documento

<article> - Contenido Independiente

El elemento <article> representa una composición auto-contenida que podría distribuirse independientemente. Piensa en artículos de periódico, entradas de blog, comentarios de usuarios, tarjetas de productos, o cualquier contenido que tenga sentido por sí solo.

Estructura de un <article> completo HTML 
<article>
  <header>
    <h2>Introducción al HTML Semántico</h2>
    <p class="article-meta">
      Publicado el <time datetime="2024-01-20">20 de enero, 2024</time>
      por <span class="author">Carlos Rodríguez</span>
    </p>
  </header>
  
  <section>
    <h3>¿Qué es HTML semántico?</h3>
    <p>El HTML semántico utiliza elementos que describen su significado...</p>
  </section>
  
  <section>
    <h3>Beneficios principales</h3>
    <p>Mejora la accesibilidad, SEO y mantenibilidad del código...</p>
  </section>
  
  <footer>
    <p class="article-tags">Etiquetas: HTML5, Semántica, Web Development</p>
    <div class="comments">5 comentarios</div>
  </footer>
</article>

¿Cuándo algo es "independiente"?

Pregúntate: "¿Este contenido tendría sentido si se publicara solo en otro sitio o en un feed RSS?"
Si la respuesta es sí, probablemente debería ser un <article>.
Ejemplos: post de blog, artículo de noticias, comentario, producto en catálogo.

Ejemplos de <article> en diferentes contextos:

Blog

Cada entrada del blog es un <article>

Periódico

Cada noticia es un <article>

Foro

Cada comentario puede ser un <article>

E-commerce

Cada producto puede ser un <article>

<aside> - Contenido Relacionado

El elemento <aside> representa contenido que está indirectamente relacionado con el contenido principal. Es contenido tangencial, como barras laterales (sidebars), pull-quotes, publicidad relacionada, biografías del autor, o enlaces relacionados.

Área Principal (Article)

Contenido principal del artículo o página

Título del Artículo Principal

Este es el contenido principal que los usuarios vienen a leer...

Área Secundaria (Aside)

Contenido relacionado indirectamente

Artículos Relacionados
  • Guía de HTML5 Semántico
  • Mejores prácticas de accesibilidad
  • Tutorial de estructura web
Implementación de <aside> en un blog HTML 
<main>
  <article>
    <h1>Guía completa de HTML Semántico</h1>
    <!-- Contenido principal del artículo -->
  </article>
  
  <aside>
    <section>
      <h3>Artículos Relacionados</h3>
      <ul>
        <li><a href="#">Introducción a CSS Grid</a></li>
        <li><a href="#">Accesibilidad Web 101</a></li>
        <li><a href="#">SEO para desarrolladores</a></li>
      </ul>
    </section>
    
    <section>
      <h3>Sobre el autor</h3>
      <p>Carlos es desarrollador web con 10 años de experiencia...</p>
    </section>
    
    <section>
      <h3>Newsletter</h3>
      <p>Suscríbete para recibir más tutoriales</p>
    </section>
  </aside>
</main>

Ubicaciones comunes de <aside>

  • Lateral: Como sidebar tradicional (a la izquierda o derecha)
  • Inline: Dentro de un artículo, para notas o información adicional
  • Después del contenido: Para contenido relacionado al final
  • Múltiples: Puedes tener varios <aside> en una página

Comparativa: ¿Cuándo usar cada uno?

Elegir entre <section>, <article> y <aside> puede ser confuso al principio. Esta tabla te ayudará a tomar la decisión correcta:

Elemento Propósito ¿Es independiente? Ejemplos comunes ¿Necesita header?
<section> Agrupar contenido temático relacionado No Capítulos, pestañas, secciones de página Sí, recomendado
<article> Contenido autocontenido e independiente Sí Posts de blog, noticias, comentarios, productos Sí, esencial
<aside> Contenido relacionado indirectamente Depende Sidebars, notas, biografías, enlaces relacionados Sí, recomendado

Diagrama de flujo para decidir:

¿El contenido es autocontenido e independiente?
Sí
Usa <article>
No
¿El contenido está tangencialmente relacionado?
Sí
Usa <aside>
No
Usa <section>

Regla mnemotécnica

Section = Separar temáticamente
Article = Aislar independientemente
Aside = Añadir relacionadamente

Jerarquía de Encabezados en Secciones

Cada elemento de sección semántica debe tener su propia jerarquía de encabezados. Esto crea un esquema del documento que ayuda a tecnologías de asistencia y mejora el SEO. La regla es simple: comienza con <h1> para el título principal, y usa encabezados consecutivos (<h2>, <h3>, etc.) para subsecciones.

<body>
<h1> Título Principal de la Página
<section>
<h2> Introducción
<article>
<h2> Título del Artículo
<section>
<h3> Subtítulo dentro del artículo
<aside>
<h2> Contenido Relacionado
Jerarquía correcta de encabezados HTML 
<body>
  <header>
    <h1>Mi Blog de Desarrollo Web</h1>
  </header>
  
  <main>
    <article>
      <h2>Guía de HTML Semántico</h2>  <!-- h2 porque está dentro de body -->
      
      <section>
        <h3>Elementos de Sección</h3>  <!-- h3 porque está dentro de article -->
        
        <section>
          <h4>El elemento <code>&lt;section&gt;</code></h4> <!-- h4 porque está anidado -->
        </section>
      </section>
    </article>
    
    <aside>
      <h2>Artículos Relacionados</h2>  <!-- h2 porque es hermano de article -->
    </aside>
  </main>
</body>

Errores comunes con encabezados

  • Saltar niveles: Ir de <h1> directamente a <h3>
  • Usar encabezados solo por estilo: Elegir <h4> porque "se ve mejor"
  • Múltiples <h1>: En HTML5 puedes tener varios, pero sigue siendo controversial
  • Secciones sin encabezados: Cada sección semántica debería tener al menos un encabezado

Anidamiento y Estructuras Complejas

Los elementos de sección semántica pueden anidarse para crear estructuras complejas y jerárquicas. El anidamiento correcto es esencial para crear un esquema de documento lógico y comprensible.

<body> - Documento completo
<main> - Contenido principal
<article> - Artículo principal
<section> - Sección 1 del artículo
<section> - Sección 2 del artículo
<aside> - Nota dentro de la sección
<aside> - Sidebar principal
<section> - Sección dentro del sidebar
Estructura anidada compleja HTML 
<!-- Foro de discusión -->
<main>
  <article>  <!-- Hilo de discusión -->
    <h2>¿Cuál es la mejor práctica para HTML semántico?</h2>
    
    <section>  <!-- Posts dentro del hilo -->
      <article>  <!-- Post individual -->
        <h3>Respuesta de Carlos</h3>
        <p>En mi experiencia, lo más importante es...</p>
        
        <aside>  <!-- Cita dentro del post -->
          <blockquote>
            Como dice la documentación oficial de MDN...
          </blockquote>
        </aside>
      </article>
      
      <article>  <!-- Otro post -->
        <h3>Respuesta de Ana</h3>
        <p>Estoy de acuerdo con Carlos, pero añadiría...</p>
      </article>
    </section>
  </article>
  
  <aside>  <!-- Sidebar del foro -->
    <section>
      <h3>Hilos populares</h3>
      <!-- Lista de hilos -->
    </section>
  </aside>
</main>

Reglas de anidamiento

  • <article> puede contener <section>, <aside>, u otros <article>
  • <section> puede contener <article>, <aside>, u otros <section>
  • <aside> puede contener <section> o <article>
  • Cualquiera puede contener elementos de contenido (p, ul, img, etc.)

Esquema del Documento (Document Outline)

Los elementos semánticos crean automáticamente un esquema del documento que los navegadores y tecnologías de asistencia usan para navegar el contenido. Este esquema es esencial para la accesibilidad y el SEO.

Sin semántica (solo divs)

body
div (contenedor)
div (contenido)
div (sidebar)
div (footer)

Esquema plano sin estructura comprensible

Con semántica

body
header (cabecera)
main (contenido principal)
article (artículo)
section (introducción)
section (desarrollo)
aside (contenido relacionado)
footer (pie)

Esquema jerárquico y semánticamente rico

Beneficios del esquema semántico

  • Accesibilidad: Los lectores de pantalla pueden navegar por secciones
  • SEO: Los motores de búsqueda entienden mejor la estructura del contenido
  • Mantenibilidad: Los desarrolladores pueden entender rápidamente la estructura
  • Navegación: Los usuarios pueden saltar rápidamente a secciones relevantes

¡Observa los detalles semánticos!

  • Anidamiento múltiple: <section> dentro de <article>, <aside> dentro de <section>
  • Jerarquía clara: Encabezados que reflejan la estructura (<h1> → <h2> → <h3> → <h4>)
  • Elementos reutilizados: <article> para el post principal y también para cada comentario
  • Estructura lógica: Cada sección tiene un propósito claro y bien definido

Patrones Comunes de Uso

Diferentes tipos de sitios web tienen diferentes necesidades estructurales. Aquí tienes patrones comunes para implementar secciones semánticas en varios tipos de sitios:

Blog o Revista

Estructura centrada en artículos con sidebar para contenido relacionado.

<main>
  <article> (post)
    <section> (secciones)
    <aside> (notas)
  </article>
  <aside> (sidebar)
</main>

E-commerce

Productos como artículos independientes, secciones para categorías.

<main>
  <section> (categoría)
    <article> (producto)
    <article> (producto)
  </section>
  <aside> (filtros)
</main>

Portal de Noticias

Múltiples artículos agrupados por secciones temáticas.

<main>
  <section> (nacional)
    <article> (noticia)
  </section>
  <section> (internacional)
    <article> (noticia)
  </section>
</main>

Sitio Educativo

Lecciones como artículos, agrupadas en secciones por tema.

<main>
  <section> (módulo)
    <article> (lección)
    <aside> (ejercicio)
    <article> (lección)
  </section>
</main>

Foro o Comunidad

Hilos como artículos, con artículos anidados para respuestas.

<main>
  <article> (hilo)
    <article> (respuesta)
    <article> (respuesta)
  </article>
  <aside> (hilos populares)
</main>

Sitio Corporativo

Secciones para diferentes áreas, artículos para casos de estudio.

<main>
  <section> (servicios)
  <section> (equipo)
  <section> (casos de éxito)
    <article> (caso)
  </section>
</main>

Checklist para elegir el patrón correcto

  • ¿El contenido principal es independiente? → <article>
  • ¿Necesitas agrupar por temas? → <section>
  • ¿Hay contenido relacionado indirectamente? → <aside>
  • ¿La estructura refleja el propósito del sitio? → Revisa patrones similares

Actividad Práctica: Convertir Divs a Secciones Semánticas

Tienes el siguiente código HTML de una página de noticias que usa solo <div>. Tu tarea es convertir todos los divs a elementos semánticos apropiados (<section>, <article>, <aside>).

Código inicial (solo divs) HTML 
<!DOCTYPE html>
<html>
<head>
    <title>Portal de Noticias</title>
</head>
<body>
    <div id="cabecera">
        <h1>Noticias del Día</h1>
        <div class="menu">
            <a href="#">Inicio</a>
            <a href="#">Nacional</a>
            <a href="#">Internacional</a>
            <a href="#">Deportes</a>
        </div>
    </div>
    
    <div id="contenido-principal">
        <div class="noticia-destacada">
            <h2>Avance científico revoluciona la medicina</h2>
            <p>Investigadores descubren nuevo tratamiento...</p>
            <div class="fecha">20 de enero, 2024</div>
            <div class="autor">Por: Dra. María López</div>
        </div>
        
        <div class="otras-noticias">
            <h3>Últimas Noticias</h3>
            <div class="noticia">
                <h4>Nueva política económica</h4>
                <p>El gobierno anuncia medidas...</p>
            </div>
            <div class="noticia">
                <h4>Resultados deportivos</h4>
                <p>El equipo local gana el campeonato...</p>
            </div>
        </div>
        
        <div class="sidebar">
            <h3>Noticias Populares</h3>
            <div class="popular">Noticia más leída</div>
            <div class="popular">Noticia viral</div>
            <div class="popular">Noticia comentada</div>
        </div>
    </div>
    
    <div id="pie">
        <div class="copyright">© 2024 Portal de Noticias</div>
        <div class="enlaces-pie">
            <a href="#">Contacto</a>
            <a href="#">Aviso Legal</a>
            <a href="#">Política de Privacidad</a>
        </div>
    </div>
</body>
</html>

Instrucciones paso a paso:

  1. Analiza cada <div>:
    • ¿Qué función cumple en la página?
    • ¿Es contenido independiente, agrupación temática o contenido relacionado?
  2. Reemplaza los divs principales:
    • div id="cabecera" → ¿Qué elemento semántico usarías?
    • div class="menu" → ¿Qué elemento semántico usarías?
    • div id="contenido-principal" → ¿Qué elemento semántico usarías?
  3. Reemplaza los divs de contenido:
    • div class="noticia-destacada" → ¿<article> o <section>?
    • div class="otras-noticias" → ¿<section> o <aside>?
    • div class="noticia" (dentro de otras-noticias) → ¿Qué elemento usarías?
    • div class="sidebar" → ¿Qué elemento semántico usarías?
    • div class="popular" → ¿Qué elemento usarías?
  4. Reemplaza los divs estructurales:
    • div id="pie" → ¿Qué elemento semántico usarías?
    • div class="enlaces-pie" → ¿Qué elemento usarías?
  5. Mejora la semántica:
    • ¿Puedes usar <time> para la fecha?
    • ¿Puedes usar <address> para información de contacto?
    • ¿Puedes agregar <header> y <footer> dentro de los artículos?
  6. Verifica tu trabajo:
    • ¿Los elementos están correctamente anidados?
    • ¿La jerarquía de encabezados es lógica?
    • ¿El código es más legible que el original?
    • ¿Incluye todos los elementos semánticos apropiados?

Solución sugerida

Pista: La noticia destacada y cada noticia en "otras-noticias" deberían ser <article> porque son contenidos independientes. "otras-noticias" como conjunto debería ser <section> para agrupar artículos relacionados temáticamente. El sidebar debería ser <aside> porque contiene contenido relacionado indirectamente.

Resumen de la lección

Los elementos de sección semántica <section>, <article> y <aside> son fundamentales para crear estructuras HTML significativas y accesibles. Cada uno tiene un propósito específico y debe usarse según la naturaleza del contenido.

<section>

Propósito: Agrupar contenido temáticamente relacionado

Regla: Debe tener un encabezado y agrupar contenido que forma parte de un todo mayor

<article>

Propósito: Contenido independiente y autocontenido

Regla: Debe tener sentido por sí solo (como en un feed RSS)

<aside>

Propósito: Contenido relacionado indirectamente

Regla: Debe ser tangencial al contenido principal (como sidebars o notas)

La jerarquía de encabezados dentro de estos elementos es crucial para crear un esquema de documento comprensible. Cada elemento de sección debe tener al menos un encabezado, y los niveles de encabezado deben reflejar la estructura anidada del contenido.

El anidamiento correcto de estos elementos permite crear estructuras complejas y significativas. Recuerda que <article> puede contener <section>, <section> puede contener <article>, y ambos pueden contener <aside> para contenido relacionado.

Al utilizar correctamente estos elementos semánticos, mejoras la accesibilidad (los lectores de pantalla pueden navegar mejor), el SEO (los motores de búsqueda entienden mejor la estructura), y la mantenibilidad (el código es más legible y comprensible para otros desarrolladores).

Regla de oro final

"Si el contenido es independiente y tendría sentido en otro contexto, usa <article>. Si es una agrupación temática dentro de un todo mayor, usa <section>. Si es contenido relacionado pero no esencial, usa <aside>."
