Layout Semántico

Objetivos de aprendizaje

  • Comprender la importancia del HTML semántico
  • Diferenciar entre elementos semánticos y no semánticos
  • Conocer los elementos semánticos de estructura principales
  • Reemplazar divs genéricos por elementos semánticos apropiados
  • Crear una estructura básica de página con semántica
  • Mejorar accesibilidad y SEO con HTML semántico

¿Qué es HTML Semántico?

El HTML semántico usa elementos que describen el significado del contenido, no solo su apariencia. En lugar de usar <div> y <span> genéricos, usamos elementos específicos como <header>, <nav>, <article>, que explican claramente qué representan.

Evolución del diseño web

1990s

Era de las tablas

Las tablas (<table>) se usaban para todo el layout. Código difícil de mantener y poco accesible.

2000s

Era de los divs

<div id="header">, <div class="nav">. Mejor separación entre contenido y estilo, pero aún sin significado semántico.

2010s+

HTML5 Semántico

<header>, <nav>, <main>, <article>. Elementos con significado intrínseco que mejoran accesibilidad, SEO y mantenibilidad.

¿Por qué importa esta evolución?

Cada etapa representó un avance en cómo pensamos la web. Las tablas eran para presentar datos, no para diseño. Los divs permitieron diseño flexible con CSS. Los elementos semánticos permiten que tanto humanos como máquinas (motores de búsqueda, lectores de pantalla) entiendan la estructura y significado del contenido.

Elementos semánticos vs no semánticos

Elementos Semánticos

Describen su significado tanto al navegador como al desarrollador.

  • <header> - Cabecera
  • <nav> - Navegación
  • <main> - Contenido principal
  • <article> - Contenido independiente
  • <footer> - Pie de página
Elementos No Semánticos

No proporcionan información sobre su contenido.

  • <div> - Contenedor genérico
  • <span> - Contenedor en línea genérico
  • <b> - Negrita (solo estilo)
  • <i> - Itálica (solo estilo)
  • <font> - Fuente (obsoleto, solo estilo)
Comparación: Semántico vs No Semántico HTML 
<!-- NO SEMÁNTICO (mal) -->
<div id="header">
  <div class="logo">Mi Sitio</div>
  <div class="menu">...</div>
</div>

<!-- SEMÁNTICO (bien) -->
<header>
  <h1>Mi Sitio</h1>
  <nav>...</nav>
</header>

Elementos de sección: <section>, <article>, <aside>

<section>

Agrupación temática

Agrupa contenido temáticamente relacionado. Cada sección debe tener su propio encabezado.

<section>
  <h2>Nuestros Servicios</h2>
  <p>Descripción de servicios...</p>
</section>

<article>

Contenido independiente

Contenido autocontenido que podría distribuirse independientemente (post de blog, noticia).

<article>
  <h2>Título del artículo</h2>
  <p>Contenido completo...</p>
</article>

<aside>

Contenido relacionado indirectamente

Contenido tangencialmente relacionado (sidebars, notas al margen, publicidad).

<aside>
  <h3>Noticias relacionadas</h3>
  <ul>...</ul>
</aside>

¿<section> o <article>?

Pregúntate: "¿Este contenido tendría sentido por sí solo en un feed RSS o similar?"
Si la respuesta es sí, usa <article>.
Si es no, pero agrupa contenido temático, usa <section>.

Estructura básica de página HTML 
<body>
  <header>
    <h1>Mi Sitio Web</h1>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <h2>Título del artículo</h2>
      <p>Contenido...</p>
    </article>
  </main>

  <footer>
    <p>© 2024 Mi Sitio Web</p>
  </footer>
</body>

¡Solo un <main> por página!

El elemento <main> debe ser único en cada página HTML. Representa el contenido principal único de la página. Si tienes múltiples elementos <main>, estarás violando las especificaciones de HTML5.

Navegación semántica: <nav>

El elemento <nav> está diseñado específicamente para bloques de enlaces de navegación. No todos los grupos de enlaces deben ir en <nav>, solo aquellos que constituyen la navegación principal del sitio.

  • Inicio
  • Acerca
  • Servicios
  • Contacto
Uso correcto de <nav> HTML 
<nav>
  <ul>
    <li><a href="index.html">Inicio</a></li>
    <li><a href="acerca.html">Acerca</a></li>
    <li><a href="servicios.html">Servicios</a></li>
    <li><a href="contacto.html">Contacto</a></li>
  </ul>
</nav>

¿Cuándo usar <nav>?

  • Sí: Menú principal de navegación, breadcrumbs, navegación entre páginas
  • No: Enlaces en el footer, enlaces sociales, enlaces dentro de un artículo
  • Regla práctica: Si los enlaces son esenciales para navegar el sitio, usa <nav>

Más elementos semánticos

<figure> y <figcaption>

Para imágenes, diagramas, código, etc., con una leyenda descriptiva.

<figure>
  <img src="imagen.jpg" alt="Descripción">
  <figcaption>Texto descriptivo de la imagen</figcaption>
</figure>

<time>

Para representar fechas y horas de forma legible por máquinas.

<time datetime="2024-01-15">15 de enero, 2024</time>
<time datetime="14:30">2:30 PM</time>

<mark>

Para resaltar texto relevante en el contexto actual (como resultados de búsqueda).

El <mark>HTML semántico</mark> mejora la accesibilidad.

<blockquote> y <cite>

Para citas textuales y referencias a fuentes.

<blockquote>
  <p>El buen código es como un buen chiste...</p>
  <cite>Programador Anónimo</cite>
</blockquote>

Datos legibles por máquinas

Elementos como <time> permiten que los navegadores, motores de búsqueda y otras herramientas procesen la información de forma inteligente. El atributo datetime proporciona la fecha en formato máquina, mientras que el contenido es para humanos.

Ejemplo práctico: Blog Semántico

header

Logo + Navegación principal

main

article

Post del blog

aside

Sidebar con widgets

footer

Info de contacto + enlaces

Código completo del blog HTML 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi Blog Semántico</title>
</head>
<body>
  <header>
    <h1>Mi Blog Personal</h1>
    <nav>
      <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="acerca.html">Acerca</a></li>
        <li><a href="contacto.html">Contacto</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>Introducción al HTML Semántico</h2>
        <p>Publicado el <time datetime="2024-01-15">15 de enero, 2024</time></p>
      </header>
      
      <section>
        <h3>¿Qué es HTML semántico?</h3>
        <p>Contenido sobre HTML semántico...</p>
        <figure>
          <img src="semantica.jpg" alt="Diagrama de HTML semántico">
          <figcaption>Estructura semántica de una página web</figcaption>
        </figure>
      </section>
    </article>

    <aside>
      <section>
        <h3>Categorías</h3>
        <ul>
          <li><a href="#">HTML</a></li>
          <li><a href="#">CSS</a></li>
          <li><a href="#">JavaScript</a></li>
        </ul>
      </section>
    </aside>
  </main>

  <footer>
    <p>© 2024 Mi Blog Personal. Todos los derechos reservados.</p>
    <address>
      Contacto: <a href="mailto:info@ejemplo.com">info@ejemplo.com</a>
    </address>
  </footer>
</body>
</html>

¡Observa los detalles!

  • Múltiples headers: Hay un <header> principal y otro dentro del <article>
  • Anidamiento: <section> dentro de <article>
  • Elementos semánticos: <time>, <figure>, <address>
  • Estructura clara: Es fácil entender la organización solo leyendo el HTML

Mejores prácticas de HTML semántico

Qué Hacer

  • Usa elementos semánticos cuando describan adecuadamente el contenido
  • Anida correctamente (ej: header dentro de article)
  • Usa múltiples nav, header, footer cuando sea apropiado
  • Incluye encabezados (<h1>-<h6>) en secciones

Qué NO Hacer

  • No uses elementos semánticos solo por su estilo por defecto
  • No abuses de <div> cuando haya elementos semánticos apropiados
  • No uses <section> como contenedor genérico para estilo
  • No crees múltiples <main> en una página

Herramientas

  • Validador HTML (validator.w3.org)
  • Herramientas de esquema (HTML5 Outliner)
  • Extensiones de navegador para visualizar semántica
  • Lectores de pantalla para probar accesibilidad

Regla de oro

"Si puedes describir lo que hace un elemento con una palabra simple (navegación, artículo, pie), usa el elemento semántico correspondiente (<nav>, <article>, <footer>)."

Actividad práctica: Convierte divs a HTML semántico

Tienes el siguiente código HTML que usa solo <div>. Convierte todos los divs a elementos semánticos apropiados.

Código inicial (con divs) HTML 
<!DOCTYPE html>
<html>
<head>
    <title>Mi Sitio Web</title>
</head>
<body>
    <div id="cabecera">
        <div class="logo">Mi Logo</div>
        <div class="menu">
            <a href="#">Inicio</a>
            <a href="#">Acerca</a>
            <a href="#">Contacto</a>
        </div>
    </div>
    
    <div id="contenido">
        <div class="articulo">
            <h2>Título del Artículo</h2>
            <p>Contenido del artículo...</p>
            <div class="fecha">Publicado el 10 de enero, 2024</div>
        </div>
        
        <div class="sidebar">
            <h3>Noticias Relacionadas</h3>
            <div class="noticia">Noticia 1</div>
            <div class="noticia">Noticia 2</div>
        </div>
    </div>
    
    <div id="pie">
        <div class="copyright">© 2024 Mi Sitio Web</div>
        <div class="enlaces">
            <a href="#">Política de Privacidad</a>
        </div>
    </div>
</body>
</html>

Instrucciones:

  1. Analiza cada <div>:
    • ¿Qué función cumple en la página?
    • ¿Hay un elemento semántico que describa mejor esa función?
  2. Reemplaza los divs:
    • div id="cabecera" → ¿Qué elemento semántico usarías?
    • div class="menu" → ¿Qué elemento semántico usarías?
    • div id="contenido" → ¿Qué elemento semántico usarías?
    • div class="articulo" → ¿Qué elemento semántico usarías?
    • div class="sidebar" → ¿Qué elemento semántico usarías?
    • div id="pie" → ¿Qué elemento semántico usarías?
    • div class="fecha" → ¿Qué elemento semántico usarías?
  3. Mejora el contenido:
    • ¿Puedes usar <time> para la fecha?
    • ¿Puedes usar <article> para cada noticia?
    • ¿Puedes usar <nav> para los enlaces del pie?
  4. Verifica tu trabajo:
    • ¿Tienes solo un <main>?
    • ¿Los elementos están correctamente anidados?
    • ¿El código es más legible que el original?
    • ¿Incluye todos los elementos semánticos apropiados?

Consejo profesional

No todo <div> debe ser reemplazado. Los <div> siguen siendo útiles para agrupaciones genéricas sin significado semántico, especialmente cuando necesitas contenedores para aplicar estilos CSS. La clave es usar elementos semánticos cuando describan adecuadamente el contenido.

Resumen de la lección

El HTML semántico usa elementos que describen el significado de su contenido, no solo su presentación. Representa una evolución desde las tablas (1990s) y los divs (2000s) hacia una web más accesible, mejor posicionada en buscadores y más fácil de mantener.

Los elementos semánticos principales incluyen <header>, <nav>, <main>, <article>, <section>, <aside>, y <footer>. Cada uno tiene un propósito específico y debe usarse cuando describe adecuadamente el contenido que contiene.

Los beneficios del HTML semántico son múltiples: mejor accesibilidad (lectores de pantalla pueden navegar mejor), mejor SEO (los motores de búsqueda entienden mejor el contenido), mejor mantenibilidad (el código es más legible y estructurado), y mejor compatibilidad futura (los estándares evolucionan hacia la semántica).

Recuerda la regla de oro: "Si puedes describir lo que hace un elemento con una palabra simple, usa el elemento semántico correspondiente." Con práctica, el uso de HTML semántico se volverá natural y transformará la forma en que construyes sitios web.
